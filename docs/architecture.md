# Architecture

## Desktop layers

- **UI pages:** status, monitor, calibration, mapping, profiles, adapter, diagnostics and settings
- **Joystick service:** SDL discovery, polling, hot-plug and virtual demo controller
- **Calibration service:** per-axis min/center/max/deadzone normalization
- **Channel mapper:** generic source mapping and safe RC pulse generation
- **Profile service:** persistent configuration and import/export
- **Serial/protocol services:** real UART, Arduino/ESP32 framing and deterministic ESP32 simulator
- **Diagnostics service:** bounded event history and export

## Firmware variants

### ESP32-S3 standalone adapter

- **USB Host/HID:** interface discovery, raw input report acquisition and generic descriptor decoding
- **Input model:** normalized axes, buttons and hats independent of joystick model
- **Profile runtime:** source mapping, reverse, trim, expo, smoothing and failsafe
- **PPM engine:** RMT waveform generation independent of USB timing
- **Protocol:** profile configuration, status and bounded desktop test stream
- **NVS:** atomic active-profile persistence

The ESP32-S3 continues operating after the desktop application disconnects because the active profile and joystick processing are local to the adapter.

### Arduino UNO/Nano desktop bridge

- **Desktop-owned input pipeline:** joystick discovery, calibration and mapping run on Windows
- **Serial stream:** final RC pulse values are sent through the shared CRC16 `SJ` protocol
- **PPM engine:** AVR Timer1 generates deterministic PPM on D9
- **Failsafe:** missing valid channel packets for 700 ms forces CH3 to 1000 µs and other channels to 1500 µs
- **Watchdog:** resets the ATmega328P when the main loop stops responding

The Arduino bridge requires the desktop application to remain running. It does not provide USB-host joystick support or persistent adapter-side profiles.

## Runtime priority

PPM waveform timing is generated by a hardware timer or RMT peripheral rather than by desktop timing. Loss of valid input always transitions to deterministic configured or firmware-default failsafe values.
